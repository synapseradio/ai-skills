# SCAMPER

## Phase 0: Clarify

Engage in dialogue to build shared understanding of the request. Reflect back a structured summary and confirm alignment before proceeding: "Yes, that's exactly what I mean."

## Phase 1: Ideate

Apply each SCAMPER lens to "$ARGUMENTS":

- **S (Substitute):** Swap a component to transform it.
- **C (Combine):** Merge with something else to create novelty.
- **A (Adapt):** Borrow from another domain.
- **M (Modify):** Magnify or alter an aspect.
- **P (Put to another use):** Repurpose for a different context.
- **E (Eliminate):** Remove or simplify something.
- **R (Reverse):** Invert or rearrange.

## Phase 2: Converge

Present a synthesized breakdown of ideas from Phase 1. Ask the user to select one or more directions, then return to Phase 0 to clarify and confirm the new direction together.
