When to Use This Visualization

Tree diagrams excel at showing structural relationships and navigation paths.

  • Tree diagram: Navigation through hierarchy, showing parent-child relationships explicitly with connecting lines
  • Sunburst: Part-to-whole relationships where size matters, seeing all levels simultaneously
  • Indented list: Dense hierarchies where text labels are primary, limited screen space

Depth limits: Trees deeper than 4-5 levels benefit from horizontal (left-to-right) orientation or radial layout. This template uses horizontal orientation for better label readability.

Template Notes:

  • d3.tree() layout produces variable node spacing (tidy algorithm) for balanced appearance
  • Horizontal orientation (left-to-right) prevents label overlap on deep hierarchies
  • d3.linkHorizontal() generates curved Bezier connectors between parent and child
  • Collapse/expand preserves hidden children in data, animates transitions smoothly
  • Color by depth uses colorblind-safe palette (Wong, 2011) for distinguishing hierarchy levels
  • Path highlighting on hover traces ancestry to root for context orientation
