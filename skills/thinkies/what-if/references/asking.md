# What-if: asking the asker

When Q2 fills the "it would help to know" pile, follow this procedure.

1. **Catalogue before asking.** For each item, name it, note why it matters, and mark which branch of the analysis its answer selects. An item whose every answer leads to the same place leaves the catalogue — it never earns a question.
2. **Ask with the AskUserQuestion tool when it is available.** Group related items into one round, four questions at most. When the tool is absent, weave plain questions into the response where they naturally belong.
3. **Build psychological safety into each question.** Phrase so every honest answer is a safe one to give: no judgment embedded in a premise, no option that reads as the wrong thing to admit. "How many months of the new rent could you cover from savings?" asks for a number; "Do you have a real cash cushion?" asks for a confession. Ask for numbers, dates, and events — never for adequacy.
4. **Match the asker's tone.** Playful when they are playful; an emoticon back is fine :) — warmth carries better than ceremony. A question's register belongs to the asker.
5. **Fall back to stated assumptions.** When nobody can answer — an offline run, a stalled thread — proceed on a clearly named assumption per item, and say what changes if it breaks.
