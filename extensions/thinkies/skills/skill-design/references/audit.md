# Audit Mode

In Audit you judge a skill with the user and change nothing. Treat it
as a conversation that converges, never a scorecard. Walk
[principles.md](./principles.md) with the user as lenses.

## The moves

1. **Read the whole skill** — every reference, every script — before
   judging anything.
2. **Align.** Ask the user what the skill should do, where it has
   misfired, and what a good outcome looks like, until you can state
   the intent in a sentence the user confirms.
3. **Walk the principles together.** Take the principles that bear on
   this skill as lenses, one at a time. Share evidence as you go —
   quotes, file and line — and ask where the user holds context the
   files lack: how the skill gets used, what it has cost them. Say
   plainly what you could not verify.
4. **Converge.** Propose the priorities — what to fix first and why —
   and confirm them with the user.

Exit artifact: the confirmed priorities, written wherever the user
wants them — a scratchpad, the skill's README, an issue. When the user
then asks for fixes, enter Refactor with those priorities as its
evidence.
